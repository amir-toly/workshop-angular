Il existe différente manière de filtrer des éléments avec AngularJS. Précédemment, les logs ont été filtré par | (pipe), où le framework nous offre la possibilité d'utiliser des filtres "built-in" ou des filtres custom.

Mais il est aussi possible dans certains cas d'utilisation de faire appel à une méthode du contrôleur pour filtrer une collection d'éléments.

Dans cette étape, il va falloir créer une méthode pour filtrer les logs par méthode et status HTTP. Cette méthode sera invoquée à chaque modification d'état des différentes checkbox définies dans la vue.

Pour se faire, il va falloir dire au framework d'écouter les modifications d'état de nos modèles dans la vue :

  • Dans le fichier workshop-angular/app/js/app.js, déclarer un objet selectedStatus attaché au $scope au sein du contrôleur ayant pour clé un status HTTP (200, 400, ...) et pour valeur un booléen à true (par défaut, tous les filtres sont sélectionnés)
  • Déclarer un objet selectedMethods attaché au $scope au sein du contrôleur ayant pour clé une méthode HTTP (GET, POST, ...) et pour valeur un booléen à true (par défaut, tous les filtres sont sélectionnés)
  • Définir une function filterLogs qui itère sur les logs du $scope et alimente une nouvelle variable selectedLogs si le log courant match les critères de filtres (par méthode et par status) sélectionnés
  • Invoquer la méthode $wathCollection sur le $scope en lui passant les objets précédemment déclarés et la méthode filterLogs en guise de callback
  • Dans le fichier workshop-angular/app/index.html, ajouter l'attribut ng-model au niveau des différentes checkbox en faisant référence à la propriété de l'objet concerné (e.g selectedMethods['GET'] pour le filtre sur la méthode GET)
